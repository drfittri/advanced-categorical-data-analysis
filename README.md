# Advanced Categorical Data Analysis

**A three-part portfolio: Ordinal, Multinomial, and Correlated-Data Logistic Regression**

This repository contains a reproducible Quarto project that extends binary logistic
regression to three real-world data structures that the standard model cannot handle:
an **ordered** outcome, an **unordered multi-category** outcome, and a **clustered
(correlated)** binary outcome. Each analysis is fully self-contained, runs a complete
diagnostic suite aligned with Hosmer, Lemeshow & Sturdivant's *Applied Logistic
Regression* (3rd ed.), and renders to a single, styled HTML document.

---

## The three analyses

| Part | Analysis | Outcome type | Method | Engine | Dataset |
|------|----------|--------------|--------|--------|---------|
| I  | Diabetes severity | Ordinal (3 ordered levels) | Proportional odds / cumulative link model | `ordinal::clm` | CDC BRFSS 2015 (stratified n = 2,200) |
| II | Contraceptive method choice | Nominal (3 unordered levels) | Multinomial logistic regression | `VGAM::vglm`, cross-checked with `nnet::multinom` | 1987 Indonesia CMC survey (n = 1,473) |
| III | Hypertension across clinics | Binary, clustered | GLMM (conditional) and GEE (population-average) | `lme4::glmer`, `gee`, `geepack` | Synthetic, 1,000 patients in 25 clinics |

**Part I** asks which sociodemographic, behavioural, and clinical factors are
associated with increasing diabetes severity. **Part II** asks which factors shape
*which* contraceptive method a woman chooses. **Part III** asks how age, BMI, physical
activity, and smoking relate to hypertension once the clustering of patients within
clinics is accounted for, and contrasts the conditional and population-average answers.

## Repository layout

```
advanced-categorical-data-analysis/
├── README.md                     # this file
├── REPRODUCE.md                  # exact versions + one-command rebuild
├── LICENSE                       # MIT licence
├── .gitignore
├── analysis/
│   ├── categorical_data_analysis_combined.qmd   # the combined report (render this)
│   ├── diabetes_sample_2200.csv                 # Part I data
│   ├── cmc.csv                                  # Part II data
│   ├── hypertension_clinics.xlsx                # Part III data
│   └── standalone/                              # the three original, separate analyses
│       ├── ordinal_logistic_regression_diabetes_munir.qmd
│       ├── multinomial_analysis_cmc_munir.qmd
│       └── correlated_binary_glmm_vs_gee.qmd
├── scripts/
│   └── generate_hypertension_data.R             # reproduces the synthetic Part III data
└── docs/                                         # rendered HTML output lands here (optional)
```

The three datasets are kept **alongside** the Quarto document in `analysis/` so that the
report's relative file paths (`read_csv("cmc.csv")`, etc.) resolve without any editing.

## Quick start

```bash
# 1. clone
git clone https://github.com/your-username/advanced-categorical-data-analysis.git
cd advanced-categorical-data-analysis

# 2. install the R packages (see REPRODUCE.md for the full list)
Rscript -e 'source("scripts/install_packages.R")'   # optional helper, or install manually

# 3. render the report
quarto render analysis/categorical_data_analysis_combined.qmd
```

The output is a single self-contained `categorical_data_analysis_combined.html` with an
embedded table of contents, folded code, and all figures and tables. See
[`REPRODUCE.md`](REPRODUCE.md) for exact software versions and the complete package list.

## Data sources & attribution

- **Part I** — CDC Behavioral Risk Factor Surveillance System (BRFSS) 2015; UCI Machine
  Learning Repository, *CDC Diabetes Health Indicators Dataset*
  (<https://doi.org/10.24432/C53919>). The file in this repo is a stratified sample.
- **Part II** — 1987 National Indonesia Contraceptive Prevalence Survey; UCI Machine
  Learning Repository, *Contraceptive Method Choice Data Set*.
- **Part III** — Synthetic data generated by `scripts/generate_hypertension_data.R` for
  teaching purposes; no real patients are involved.
